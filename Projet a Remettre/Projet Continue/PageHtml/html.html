Balise html

Accueil

Liste de balise

1.‹!Doctype html›

Permet au navigateur de faire la différence entre la dernière version Html et l'ancien Html.

Ex: !Doctype

2.‹html›

Enveloppe tout le contenu HTML du document.

Ex: html

3.‹head›

Enveloppe les métadonnées de la page Web. Les métadonnées décrivent des données sur notre page Web.
Vous devez inclure un titre dans le navigateur principal de l'onglet.

Ex: head

4.‹body›

La balise ‹body› entoure le contenu comme du texte, des images, etc.
Cela sera visible sur la page Web.

Ex: body

5.‹footer›

Il définit le contenu qui sera affiché tout en bas de notre page Web sous tous nos autres contenus.

Ex: footer

6.‹meta›

Les métadonnées sont des données qui décrivent des données,
et le langage HTML a une manière « officielle » d'ajouter des métadonnées à un document

Ex: meta

7.‹title›

définit le titre du document qui s'affiche dans la barre d'outils du navigateur

Ex: title

8.‹h1› ,‹h2›, ‹h3›, ‹h4›, ‹h5›, ‹h6›

HTML définit six niveaux de titres. Un élément de titre implique tous les changements de police,
les sauts de paragraphe avant et après, et tout espace blanc nécessaire pour rendre le titre.
Les éléments de titre sont H1, H2, H3, H4, H5 et H6, H1 étant le niveau le plus élevé (ou le plus important) et H6 le moins important.

Ex: headers

9.‹p›

Produira un texte sur une seule ligne sauf si des sauts ‹br› sont spécifiés.

Ex: ‹p›

10.‹Div›

La balise ‹div› définit une division ou une section dans un document HTML.
La balise ‹div› est utilisée comme conteneur pour les éléments HTML - qui sont ensuite stylisés avec CSS ou manipulés avec JavaScript.
La balise ‹div› est facilement stylée en utilisant l'attribut class ou id.
N'importe quel type de contenu peut être placé à l'intérieur de la balise ‹div›.

Ex: ‹Div›

11.‹Span›

La balise ‹span› est un conteneur en ligne utilisé pour marquer une partie d'un texte ou une partie d'un document.
La balise ‹span› est facilement stylisée par CSS ou manipulée avec JavaScript à l'aide de l'attribut class ou id.
La balise ‹span› ressemble beaucoup à l'élément ‹div›, mais ‹div› est un élément de niveau bloc et ‹span› est un élément en ligne.

Ex: ‹Span›

12.‹i›

La balise ‹i› définit une partie de texte dans une voix ou une humeur alternative.
Le contenu à l'intérieur est généralement affiché en italique.
La balise ‹i› est souvent utilisée pour indiquer un terme technique, une phrase d'une autre langue, une pensée, un nom de navire, etc.
Utilisez l'élément ‹i› uniquement lorsqu'il n'existe pas d'élément sémantique plus approprié, tel que :

‹em› (texte mis en évidence)
‹strong› (texte important)
‹marque › (texte marqué/surligné)
‹cite› (le titre d'une œuvre)
‹dfn› (un terme de définition)

Ex: ‹I›

13.‹img›

La balise ‹img› est utilisée pour intégrer une image dans une page HTML.
Les images ne sont pas techniquement insérées dans une page Web;
les images sont liées à des pages Web. La balise ‹img› crée un espace de maintien pour l'image référencée.
La balise ‹img› a deux attributs obligatoires :
src - Spécifie le chemin vers l'image
alt - Spécifie un texte alternatif pour l'image, si l'image ne peut pas être affichée pour une raison quelconque
Remarque : Indiquez également toujours la largeur et la hauteur d'une image. Si la largeur et
la hauteur ne sont pas spécifiées, la page peut scintiller pendant le chargement de l'image.

Astuce : Pour lier une image à un autre document, il suffit d'imbriquer la balise ‹img› dans une balise ‹a› (voir l'exemple ci-dessous).

Ex: ‹Img›

14.‹li›

La balise ‹li› définit un élément de liste.
La balise ‹li› est utilisée dans les listes ordonnées (‹ol›), les listes non ordonnées (‹ul›)
et dans les listes de menus (‹menu›).
Dans ‹ul› et ‹menu›, les éléments de la liste seront généralement affichés avec des puces.
Dans ‹ol›, les éléments de la liste seront généralement affichés avec des chiffres ou des lettres.

Conseil : utilisez CSS pour styliser les listes.

Ex: ‹li›

15.‹input type= "text"›

La balise ‹input› spécifie un champ de saisie dans lequel l'utilisateur peut saisir des données.
L'élément ‹input› est l'élément de formulaire le plus important.
L'élément ‹input› peut être affiché de plusieurs manières, selon l'attribut type.

Ex: ‹input›

16.‹ol›

La balise ‹ol› définit une liste ordonnée. Une liste ordonnée peut être numérique ou alphabétique.
La balise ‹li› est utilisée pour définir chaque élément de la liste.

Conseil : utilisez CSS pour styliser les listes.
Conseil: pour une liste non ordonnée, utilisez la balise ‹ul›.

Ex: ‹ol›

17.‹pre›

La balise ‹pre› définit le texte préformaté. Le texte d'un élément ‹pre› est affiché dans une police à largeur fixe,
et le texte conserve à la fois les espaces et les sauts de ligne.
Le texte sera affiché exactement tel qu'il est écrit dans le code source HTML.

Regardez aussi :

Description de la balise
‹code› Définit un morceau de code informatique
‹samp› Définit un exemple de sortie d'un programme informatique
‹kbd› Définit la saisie au clavier
‹var› Définit une variable

Ex: ‹pre›

18.‹a›

La balise ‹a› définit un lien hypertexte, qui est utilisé pour relier une page à une autre.
L'attribut le plus important de l'élément ‹a› est l'attribut href, qui indique la destination du lien.
Par défaut, les liens apparaîtront comme suit dans tous les navigateurs:

Un lien non visité est souligné et bleu
Un lien visité est souligné et violet
Un lien actif est souligné et rouge

Conseils et remarques:

Conseil: Si la balise ‹a› n'a pas d'attribut href, il s'agit uniquement d'un espace réservé pour un lien hypertexte.
Conseil: Une page liée s'affiche normalement dans la fenêtre actuelle du navigateur, sauf si vous spécifiez une autre cible.
Astuce : Utilisez CSS pour styliser les liens : Liens CSS et Boutons CSS.

Ex: ‹a›

19.‹dl›

La balise ‹dl› définit une liste de description.
La balise ‹dl› est utilisée conjointement avec ‹dt› (définit les termes/noms) et ‹dd› (décrit chaque terme/nom).

Ex: ‹dl›

20.‹dt›

La balise ‹dt› définit un terme/nom dans une liste de description.
La balise ‹dt› est utilisée conjointement avec ‹dt› (définit une liste de description) et ‹dt› (décrit chaque terme/nom).

Ex: ‹dt›

21.‹dd›

La balise ‹dd› est utilisée pour décrire un terme/nom dans une liste de description.
La balise ‹dd› est utilisée conjointement avec ‹dl› (définit une liste de description) et ‹dt› (définit les termes/noms).
Dans une balise ‹dd›, vous pouvez mettre des paragraphes, des sauts de ligne, des images, des liens, des listes, etc.

Ex: ‹dd›

22.‹strong›

La balise ‹strong› est utilisée pour définir le texte avec une forte importance.
Le contenu à l'intérieur est généralement affiché en gras.

Astuce: Utilisez la balise ‹b› pour spécifier du texte en gras sans aucune importance supplémentaire!

Ex: ‹strong›

23.‹nav›

La balise ‹nav› définit un ensemble de liens de navigation.
Notez que tous les liens d'un document ne doivent PAS être à l'intérieur d'un élément ‹nav›
L'élément ‹nav› est destiné uniquement aux principaux blocs de liens de navigation.
Les navigateurs, tels que les lecteurs d'écran pour les utilisateurs handicapés,
peuvent utiliser cet élément pour déterminer s'il faut omettre le rendu initial de ce contenu.

Ex: ‹nav›

24.‹ul›

La balise ‹ul› définit une liste non ordonnée (à puces).
Utilisez la balise ‹ul› avec la balise ‹li› pour créer des listes non ordonnées.

Conseil : utilisez CSS pour styliser les listes.
Conseil: pour les listes ordonnées, utilisez la balise ‹ol›.

Ex: ‹ul›

25.‹b›

La balise ‹b› spécifie un texte en gras sans aucune importance supplémentaire.
Conseils et remarques:

Remarque: Selon la spécification HTML5, la balise ‹b› doit être utilisée en DERNIER recours lorsqu'aucune autre balise n'est plus appropriée.
La spécification stipule que les titres doivent être indiqués avec les balises ‹h1› à ‹h6›,
le texte mis en évidence doit être indiqué avec la balise ‹em›, le texte important doit être indiqué avec la balise ‹strong›
et le texte marqué/surligné doit être indiqué par la balise ‹mark›.

Conseil: Vous pouvez également utiliser le CSS suivant pour définir le texte en gras: "font-weight: bold;".

Ex: ‹b›

26.‹table›

La balise ‹table› définit un tableau HTML.
Un tableau HTML se compose d'un élément ‹table› et d'un ou plusieurs éléments ‹tr›, ‹th› et ‹td›.
L'élément définit une ligne de tableau, l'élément ‹th› définit un en-tête de tableau et l'élément ‹td› définit une cellule de tableau.
Un tableau HTML peut également inclure des éléments ‹caption›, ‹colgroup›, ‹thead›, ‹tfoot› et ‹tbody›.

Ex: ‹table›

27.‹table›

La balise définit un tableau HTML.
Un tableau HTML se compose d'un élément ‹table› et d'un ou plusieurs éléments ‹tr›, ‹th› et ‹td›.
L'élément définit une ligne de tableau, l'élément ‹th› définit un en-tête de tableau et l'élément ‹td›
définit une cellule de tableau.
Un tableau HTML peut également inclure des éléments ‹caption›, ‹colgroup›, ‹thead›, ‹tfoot› et ‹tbody›.

Ex: ‹table›

28.‹th›

La balise ‹th› définit une cellule d'en-tête dans un tableau HTML.
Un tableau HTML a deux types de cellules:
Cellules d'en-tête - contient des informations d'en-tête (créées avec l'élément ‹th›)
Cellules de données - contient des données (créées avec l'élément ‹th›)
Le texte des éléments ‹th› est en gras et centré par défaut.
Le texte des éléments ‹th› est régulier et aligné à gauche par défaut.

Ex: ‹th›
